Sistema de Ventas de Plantas y Macetas
📌 Descripción del Proyecto
Este proyecto consiste en el desarrollo de un sistema básico de ventas de plantas y macetas, en el cual se aplican diferentes patrones de diseño de la Programación Orientada a Objetos (POO).
El objetivo principal es demostrar cómo estos patrones permiten mejorar la organización del código, facilitar su mantenimiento y hacerlo más flexible ante futuros cambios.
🎯 Objetivos
Aplicar patrones de diseño en un caso real
Comprender la utilidad de cada patrón
Mejorar la estructura y calidad del código
Desarrollar buenas prácticas de programación
🧩 Patrones de Diseño Implementados
🏗️ 1. Singleton
Se utiliza para garantizar que solo exista una única instancia del sistema durante la ejecución del programa.
Esto permite controlar el acceso a recursos compartidos y evitar duplicaciones innecesarias.
🏭 2. Factory Method
Este patrón se implementa para la creación de objetos del tipo producto (Planta o Maceta).
Permite desacoplar la lógica de creación del resto del sistema, facilitando la extensión del código.
🎯 3. Strategy
Se utiliza para gestionar diferentes métodos de pago, como efectivo o tarjeta.
Permite cambiar el comportamiento del sistema en tiempo de ejecución sin modificar su estructura.
👀 4. Observer
Este patrón permite notificar automáticamente a los clientes cuando se realiza una venta.
Facilita la comunicación entre objetos sin generar dependencias fuertes.
🧩 5. Decorator
Se utiliza para agregar funcionalidades adicionales a los productos, como incluir fertilizante u otros extras.
Permite extender el comportamiento de los objetos de forma dinámica.
